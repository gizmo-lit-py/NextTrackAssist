# NextTrackAssist 引き継ぎメモ

## プロジェクト概要
- DJの次曲選択を支援するスコアリング型Webアプリ
- GitHub: https://github.com/gizmo-lit-py/NextTrackAssist
- ライブデモ: https://nexttrackassist-production.up.railway.app

## 技術スタック
- Python 3.11 / Flask 3.x（App Factory + Blueprint）
- SQLAlchemy 2.0 / PostgreSQL 15
- Docker / Docker Compose（dev・prod構成あり）
- Gunicorn + Nginx（SSL終端）
- pytest（SQLite in-memory）/ GitHub Actions CI
- Railway（本番デプロイ済み）

## ディレクトリ構成のポイント
```
app/
  __init__.py      # create_app() ファクトリ
  extensions.py    # engine・Sessionの定義（DATABASE_URLでSQLite/PG切替）
  config.py        # os.getenv("DATABASE_URL", ...) で環境変数読み込み
  models/          # user.py, track.py（CHECK制約あり）
  routes/          # auth_bp / tracks_bp / api_bp
  services/        # score.py, rekordbox.py, set_generator.py（ドメイン層）
scripts/
  migrate.py       # Docker起動時にBase.metadata.create_all()を実行
migrations/
  001_initial.py
  002_bpm_float.py
  003_key_constraint.py   ← コミット済み（これがなくてRailwayが落ちてた）
```

## 解決済みのバグ・トラブル

### ① ModuleNotFoundError: No module named 'app'
- 原因: `python scripts/migrate.py` 実行時にsys.path[0]が `/app/scripts/` になる
- 対処: `Dockerfile` に `ENV PYTHONPATH=/app` を追加
- コミットメッセージ: "fix: PYTHONPATH=/app を追加してモジュール解決エラーを修正"

### ② psycopg2.OperationalError: connection to server at "localhost" failed
- 原因: `DATABASE_URL` 未設定 → デフォルトのlocalhost PostgreSQLに繋ごうとする
- 対処: Railwayに PostgreSQL サービスを追加 → NextTrackAssist の Variables に `DATABASE_URL = ${{Postgres.DATABASE_URL}}` を設定して再デプロイ

### ③ migrations/003_key_constraint.py が GitHub になかった
- 原因: ローカルに存在したがコミットされていなかった
- 対処: コミット & プッシュ

## Dockerfile（最終版）
```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
ENV PYTHONPATH=/app
CMD python scripts/migrate.py && gunicorn -b 0.0.0.0:${PORT:-8000} "app:create_app()"
```
